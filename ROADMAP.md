# Roadmap

Cloudflare Engineering OS should grow like a product, not a random notes repository.

## Current status

**v0.1 Foundation** is focused on beginner onboarding, core guidance, first project playbooks, debug help, and safe update design.

## v0.2 — Beginner Project Engine

Goal: when someone types a simple project idea, the repo helps them understand the next steps.

### Planned

- More beginner project playbooks
  - AI Chat App
  - File Manager
  - Travel Agency Website
  - E-commerce Starter
  - Job Portal
  - Course/LMS site
- More prompt packs
  - Review project
  - Migrate to Cloudflare
  - Cost check
  - Security check
- Starter example folders
  - Secure Mini CMS
  - Simple Workers API
  - D1 CRUD example
  - R2 upload example
- Better navigation from README
- Beginner videos/script outline later

## v0.3 — Cloudflare Product Catalog

Goal: explain every important Cloudflare product in simple and engineering levels.

Each product page should include:

- Simple explanation
- When to use it
- When not to use it
- Beginner example
- Production notes
- Debug notes
- Related products

## v0.4 — Production Assistant

Goal: help developers prepare real projects for production.

Planned:

- Production readiness reports
- Security checklist
- Performance checklist
- Cost checklist
- Deployment checklist
- Rollback checklist
- Observability checklist

## v0.5 — Living Knowledge Engine

Goal: keep guidance current with Cloudflare changes.

Planned:

- Official source watcher improvements
- Changelog summarizer
- Draft PR generator
- Source checked date metadata
- Risk labels for Cloudflare updates

## Long-term vision

Someone should be able to say:

> I need to build a project.

And this repo should guide them through:

1. Small version 1
2. Cloudflare tools
3. Architecture
4. Database
5. Code structure
6. Local setup
7. Debugging
8. Deployment
9. Production readiness
10. Future upgrades

The standard is simple:

> Save solo developers months of confusion and help them build safely on Cloudflare.
