# Learning Sync Workflow

Use this after applying Cloudflare Engineering OS / DeveloperB guidance to a real project.

## Purpose

The main repo should learn from real project work, but not automatically trust every lesson.

Real workflows can reveal:

- Confusing instructions
- Missing migration steps
- Cloudflare limits
- AI hallucination patterns
- Messy workspace problems
- Better folder structures
- Better debug playbooks
- Better prompts

## Safe learning rule

> Capture everything. Approve carefully.

Do not automatically change the main repo based on one project.

## What to record

```text
Project name:
Project type:
Original stack:
Target Cloudflare stack:
Date:

What worked:
- 

What failed:
- 

What confused the AI agent:
- 

What confused the human developer:
- 

What files/folders became messy:
- 

What Cloudflare issue appeared:
- 

What should be improved in the repo:
- 
```

## Sync process

1. Finish a migration or build slice.
2. Record lessons in this format.
3. Remove private project details.
4. Convert lessons into a small repo improvement.
5. Open a PR.
6. Review before merge.

## Privacy rule

Before syncing lessons back:

- Remove client names if private
- Remove secrets
- Remove private URLs
- Remove personal data
- Remove business-sensitive details

## Improvement targets

Lessons can update:

- Migration guides
- Debug playbooks
- Prompt packs
- Project templates
- Folder structure guidance
- Professional checklist
- AI task discipline rules
- Cloudflare decision engine

## AI agent instruction

After completing real project work, summarize reusable lessons without exposing private project data.
