# Project Principles

These principles guide every file in Cloudflare Engineering OS.

## 1. Start with the human

A beginner should understand the first page without needing a senior developer beside them.

## 2. Version 1 must be small

Build the smallest useful product first. Add advanced features only after version 1 works.

## 3. Choose tools by need

Do not add a Cloudflare product because it is interesting. Add it because the project needs it.

## 4. Explain the why

Every recommendation should explain the reason and trade-off.

## 5. Make AI safer

AI agents should inspect, plan, and verify before changing code.

## 6. Protect production

Production data, secrets, domains, and deployments deserve careful steps and rollback plans.

## 7. Debug calmly

Most errors have a simple cause. Check bindings, secrets, migrations, environment, and commands before rewriting code.

## 8. Keep Cloudflare knowledge alive

Cloudflare changes. Updates should come from official sources and arrive as reviewable changes.

## 9. Teach through real projects

People learn faster from real project paths: CMS, news portal, CRM, marketplace, travel agency, AI chat, and file manager.

## 10. Save time for future builders

The best guide is one that saves someone from repeating the same painful mistake.
