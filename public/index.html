RELEASE PATH

Deploy to Cloudflare Workers

The root `wrangler.jsonc` deploys the Worker and Static Assets as one unit.

  1. 1
    Connect the repository

    Use the connected Cloudflare Worker Git deployment and select the `main` branch when ready.

  2. 2
    Set build command

    Run npm install; deploy with npx wrangler deploy if the dashboard requires an explicit deploy command.

  3. 3
    Check preview routes

    Open the workspace, health endpoint, and workspace API before promoting.

  4. 4
    Keep data services deferred

    Do not add D1, auth, or secrets until the corresponding user journey is approved.
